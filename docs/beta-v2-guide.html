OVERVIEW
What is SourceKit?

SourceKit is a technical sourcing tool that finds software engineers based on their actual code contributions, not self-reported keywords or pedigree. It breaks down the role and your team's DNA, then builds a sourcing strategy with specific repos to mine, target companies, and EEA criteria (Evidence of Exceptional Ability: verifiable proof someone is top 5-10%). Candidates are scored 0-100 and hidden gems are surfaced. v2.1 adds Exa Research for AI-powered strategy generation, Exa Answer for grounded candidate summaries, Parallel deep research for company intelligence, and findSimilar/findAll for expanding your pipeline from a single strong match.

Evidence Over Keywords
Candidates are scored on real GitHub activity and verified against natural-language EEA criteria using Exa's vector search. Criteria like "shipped a product in the robotics space" match against actual web content, not keywords.
AI-Built Strategy
Exa Research generates the sourcing strategy from the role spec. Claude and Parallel.ai provide fallback intelligence. The strategy includes repos, companies, skills, and EEA signals.
Minutes, Not Days
Job description to scored candidate pipeline in under two minutes. No boolean strings. No manual sourcing. No tool switching.

GETTING STARTED
Three Ways to Start a Search
1. Role + Company
Type a job title and target company. Exa Research builds the sourcing strategy: target repos, poach companies, skills matrix, and EEA criteria. Example: "Staff ML Engineer" at "Anthropic"
2. Paste a Job Description
Copy and paste the full JD text. The AI parses requirements, technologies, and seniority level to generate a targeted strategy.
3. Paste a Job URL
Drop in a link from Lever, Greenhouse, or Ashby. Parallel.ai extracts the JD content and builds the strategy automatically. Works with JS-rendered pages.

Deep Research and Intelligence APIs

v2.1 adds six API integrations: Exa Research, Exa Answer, Exa findSimilar, Parallel Company Intel, Parallel Talent Mapping, and Webset CSV import with webhooks.

Exa Research API
Strategy generation is now powered by Exa's async research endpoint. Produces structured sourcing strategies with repos, companies, skills, and EEA criteria grounded in web evidence. Claude serves as automatic fallback.
Exa Answer API
Top 10 search results are enriched with grounded summaries and citations. Each candidate gets a web-verified summary with links to the actual pages that confirm their work, not AI-generated guesses.
Exa findSimilar
Click "Find Similar" on any profile to discover engineers with similar GitHub activity and web presence. Expands your pipeline from a single strong match.
Parallel Company Intel
Deep research on target companies: estimated engineering headcount, tech stack signals, hiring and attrition patterns, and a rationale for why to source from each company. Runs in parallel across all poach targets.
Parallel Talent Mapping
"Map Engineers" on any poach company to discover specific engineers with name, title, LinkedIn, GitHub, and notable work. Uses Parallel's FindAll API for systematic web-wide discovery.
Webset CSV Import + Webhooks
Import candidate lists via CSV directly into Exa Websets for enrichment. Register webhooks to receive real-time notifications when new candidates match your Webset criteria.
SCORING
How Candidates Are Scored

Every candidate receives a composite score from 0 to 100 based on actual GitHub activity, weighted by relevance to the target role. Top 10 candidates also receive grounded summaries with web citations from the Exa Answer API.

Strong Match 80-100
Worth Reviewing 60-79
Lower Signal 0-59
Commit Volume and Recency
How actively and recently the candidate has contributed to relevant repos. Recent activity is weighted more heavily.
Language and Stack Match
How well the candidate's primary languages align with what the role requires.
Grounded Summary
Top candidates get a web-verified summary with citations from Exa Answer. Shows actual evidence of their work, not just self-reported claims.

EXA WEBSETS
Persistent Candidate Collections

Websets are persistent, auto-updating candidate collections powered by Exa's neural search. Define your criteria once and get new matches continuously. v2.1 adds CSV import and webhook support.

Create and Search
Build a Webset from any search or from scratch. Define entity type, search query, and filtering criteria. Exa finds matching candidates across the web.
Auto-Enrich
Every item is automatically enriched with current company, role, LinkedIn URL, GitHub profile, and primary programming languages.
Monitor
Set a schedule and Websets auto-update with new matches. New candidates matching your criteria appear without re-running searches.
CSV Import NEW
Upload a CSV of candidates directly into a Webset for Exa enrichment. Bulk-import your existing candidate lists and let Exa verify and enrich each one.
Webhooks NEW
Register webhook endpoints to receive real-time notifications when new candidates are discovered or existing items are updated by Exa.

BETA TESTING
What to Test
1. Sign In with Google at getsourcekit.vercel.app
2. Build a Strategy using all three input paths: role + company, pasted JD, job URL. Note the research_source field (Exa or Claude).
3. Edit and Run the Search. Review the repo list. Add, remove, rearrange, then run.
4. Check Grounded Summaries. Top 10 candidates should show italic summaries with citation links below their bio.
5. Find Similar Candidates. Open a detail view, click "Find Similar Candidates," review the results panel.
6. Map Engineers. In the strategy editor, hover a poach company and click the people icon.
7. Test Exa Websets. Create a Webset, add criteria, check enrichment, try CSV import, set monitoring.
8. Pipeline and Bulk Actions. Add candidates, drag between stages, select multiple for compare/outreach.
9. Export to CSV. Verify the data is clean and complete.
PRO TIPS
Best Practices
Be Specific With Titles
"ML Infrastructure Engineer focused on training pipelines" produces far better results than "ML Engineer." Specificity drives repo targeting.
Always Edit the Repo List
AI suggestions are a starting point. Your edits to the repo list are the single biggest lever for result quality.
Use Find Similar to Expand
One strong match + Find Similar = a cluster of similar engineers. Often the fastest way to build a shortlist.
Map Engineers Before Searching
Click "Map Engineers" on poach companies to discover specific names. Targeted leads before the broader search runs.
Websets for Ongoing Roles
Roles you hire repeatedly: set up a Webset with monitoring. Import existing candidate lists via CSV. Exa enriches and discovers new matches automatically.
Check Grounded Summaries
Italic summaries with citations on top candidates are web-verified. Click citation links to validate claims before outreach.
